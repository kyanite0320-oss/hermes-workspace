## 1. 结构完整性

- [ ] 所有映射规则涉及的目标文件都存在
- [ ] 所有映射规则涉及的目标Sheet都存在
- [ ] 目标表的列名（Row 7 内部字段名）与映射配置一致
- [ ] 输出文件行数在合理范围内（不空表、不多出一倍）

## 2. 数值一致性

- [ ] 源表 → 目标表：随机抽 3 行，逐字段比对值一致
- [ ] 公式转数值的字段，计算结果与手动演算一致（如 `J2*6` = HP×6）
- [ ] 转毫秒的字段，数值合理（不在 0~1 也不在 1000000+ 的异常范围）

## 3. 数据质量

- [ ] 不应为空的字段没有空值
- [ ] 数值列没有混入文本（没有残留的 "秒"、"像素" 等单位）
- [ ] 没有残留的公式文本（无 `ROUNDUP`、`J2*6` 等字符串）

## 4. 跨表关联

- [ ] 怪物→技能的 skill 关联：所有怪物引用的 skill ID 在 skill 表中都存在
- [ ] 逆关联检查：skill 表中被引用的行都有对应怪物

## 5. 变更摘要

- [ ] 本次处理了多少条数据（增/改）
- [ ] 列一下实际变更的值（只列出值有变化的字段，不变的跳过）
