# 地堡防线 · XM 设定会议记录

> 日期：2026-06-01
> 参与人：Kyanite (策划) / Hermes

---

## 一、讨论议题与结论

### 1. 全局设定框架确认

**讨论：** 确定全局设定的覆盖范围（世界观+系统框架），以及颗粒度（框架级，不做太细）。

**结论：** 7 大分支框架确认：
- 总览 / 战斗 / 丧尸 / 关卡 / 养成 / 经营 / 经济

### 2. 游戏命名

**讨论：** 游戏名方向。

**结论：** 「地堡防线」

### 3. 世界观方向

**讨论：** 世界观走哪个方向。

**结论：** 末世废土，氛围层不绑剧情。

### 4. 创新方向确认

**讨论：** 养成为主/战斗为主还是全都要。

**结论：** 帕鲁like，捕捉僵尸双身份（基地生产 + 战斗助战），战斗全自动。

### 5. 脑图一二级结构定稿

**讨论：** 用户修改了初始框架，确认最终结构。

**结论：** 最终结构见 XMIND 文件。

### 6. 战斗系统展开

**讨论：** 射击逻辑、技能释放、丧尸助战方式、肉鸽构筑深度。

**结论：** 自动射击有子弹类型/目标逻辑；技能冷却自动释放；丧尸对冲式助战（城堡战争like）；构筑深度链：主动技能→强化→联动→终极。

### 7. 捕捉道具设计

**讨论：** 3档命名，末世地堡风格。

**结论：**

| 档位 | 名称 | 说明 |
|------|------|------|
| 普通 | 麻醉弹 | 基础捕捉 |
| 进阶 | 震荡束缚弹 | 震荡波+钢索 |
| 高级 | 力场收容核 | 微型力场发生器 |

### 8. 捕捉小游戏

**讨论：** 手动捕捉的具体交互形式。

**结论：** 节奏判定（滑块左右滑动，指定区域按下）。

### 9. PVE 副本

**讨论：** 是否需要设计独立PVE副本。

**结论：** 暂空，非核心玩法。

### 10. 伙伴定位

**讨论：** 伙伴与丧尸的功能区分。

**结论：** 伙伴 = 防线内站桩自动射击单位（类似向僵尸开炮佣兵）；丧尸 = 冲出去对冲。技能全自动不可控。

### 11. 地脉回路 vs 装备

**讨论：** 两套装备系统的关系。

**结论：** 两套并存，装备前期→地脉回路后期接棒。

### 12. 活力系统存废

**讨论：** 活力系统是否保留。

**结论：** 取消活力。丧尸消耗闭环由"进化→变异→追完美词条"驱动。

### 13. 阵地建设定位

**讨论：** 阵地与战斗的交互方式、维护逻辑。

**结论：** 战前配置→关卡内自动生效→战后自动修复，无需手动维护。

### 14. 资源分类

**讨论：** 经济系统资源类型。

**结论：**

| 类型 | 资源 |
|------|------|
| 经营资源 | 电力（占用逻辑） |
| 基础资源 | 石料、矿石（自动生产） |
| 特殊资源 | 补给（体力） |
| 基础货币 | 铁块 |
| 高级货币 | 瓶盖（氪金） |

### 15. 自动捕捉

**讨论：** 自动捕捉的获取方式。

**结论：** 建筑功能，满足条件解锁，不设为付费独占。

### 16. 丧尸攻城

**讨论：** "特色"中的丧尸攻城是什么。

**结论：** 包含关卡（预设系统防线）和 PVP（其他玩家防线数据）两种。

### 17. 全局科技分支

**讨论：** 是否需要加更多科技分支。

**结论：** 可以加，用户自填。
